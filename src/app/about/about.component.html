About

Overview

Purpose

Chef server is a storage of cookbooks and recipes for chef client, which automates infrastructure deployment and prevents system configureation drift

Connections

  • Port 8443: HTTPS for angular frontend
  • Port 8889: HTTPS for chef-zero backend
  • Port 22: SSH

Bootstraping new nodes

After SSH to this server, go to /home/pi/chef-nodes, there create a new directory for the node you want to bootstrap, copy a file from an existing folder, make necessary changes to it.

What is needed on the new node is open SSH port (22) and a sudo access

Administration

Backend binaries

Backend binaries are a ruby project chef-zero from github, so it is a good idea to perform git pull on master branch every once in a while. The binaries location is /home/pi/git/chef-zero

Web server is a standard nginx installation

Startup

Startup script that contains the command for starting up the chef-zero backend is in /etc/rc.local and the web server is NGINX with configuration located in /etc/nginx
